## Overview

The `shared` module contains all platform-agnostic business logic, data models, and API communication for the FKS Trading Platform. This module is built with Kotlin Multiplatform (KMP) and is shared across all client applications (Android, iOS, Desktop, Web).

## Architecture

```
shared/
├── src/
│   ├── commonMain/          # Shared code for all platforms
│   │   └── kotlin/xyz/fkstrading/shared/
```

## API Client

### Interface: `ApiClient`

The main interface for all backend communication:

```kotlin
interface ApiClient {
    // Signals
    suspend fun getSignals(): Result<List<Signal>>
    suspend fun getSignalById(signalId: String): Result<Signal>
    suspend fun generateSignal(symbol: String, strategyId: String): Result<Signal>
    
    // Orders
    suspend fun getOrders(status: String?): Result<List<Order>>
    suspend fun submitOrder(...): Result<Order>
    suspend fun cancelOrder(orderId: String): Result<Order>
    
    // Positions
    suspend fun getPositions(): Result<List<Position>>
    suspend fun closePosition(positionId: String): Result<Position>
    
    // Strategies
    suspend fun getStrategies(): Result<List<Strategy>>
    suspend fun saveStrategy(strategy: Strategy): Result<Strategy>
    
    // Dashboard & Analytics
    suspend fun getDashboardOverview(): Result<DashboardResponse>
    suspend fun getPortfolioSummary(): Result<PortfolioSummary>
    
    // Health
    suspend fun getHealthStatus(): Result<HealthResponse>
    suspend fun ping(): Boolean
}
```

### Implementation: `ApiClientImpl`

Uses Ktor HTTP client with:
- Content negotiation (JSON)
- Automatic retry on failures
- Comprehensive error handling
- Request/response logging
- Timeout configuration

### Error Handling

All API calls return `Result<T>` for safe error handling:

```kotlin
val result = apiClient.getSignals()
result.onSuccess { signals ->
    // Handle success
}.onFailure { error ->
    when (error) {
        is ApiException.NetworkException -> // Network error
        is ApiException.ServerException -> // Server error (5xx)
        is ApiException.ClientException -> // Client error (4xx)
        is ApiException.TimeoutException -> // Request timeout
    }
}
```

## Configuration

### Environment Variables

Set these via Koin properties or build configuration:

- `API_BASE_URL` - Backend API base URL (default: `http://localhost:8000`)
- `WS_BASE_URL` - WebSocket base URL (default: `ws://localhost:8000`)
- `ENABLE_HTTP_LOGGING` - Enable HTTP logging (default: `true`)

### Example: Android

```kotlin
// In build.gradle.kts
android {
    defaultConfig {
        buildConfigField("String", "API_BASE_URL", "\"https://api.fkstrading.xyz\"")
    }
}

// In Application class

    properties(mapOf(
        "API_BASE_URL" to BuildConfig.API_BASE_URL
    ))
}
```

## Adding New Features

### Adding a New Domain Model

1. Create the model in `domain/models/YourModel.kt`
2. Add validation methods
3. Add helper functions
4. Create DTO in `data/models/ApiResponses.kt`
5. Add conversion extension: `fun YourModelDto.toDomain(): YourModel`
6. Write tests in `commonTest/domain/models/YourModelTest.kt`

### Adding a New API Endpoint

1. Add method signature to `ApiClient` interface
2. Implement in `ApiClientImpl`
3. Add error handling
4. Write integration test
5. Update documentation

Example:
```kotlin
// 1. Interface
interface ApiClient {
    suspend fun getBacktestResults(strategyId: String): Result<BacktestResult>
}

// 2. Implementation
class ApiClientImpl(private val httpClient: HttpClient) : ApiClient {
    override suspend fun getBacktestResults(strategyId: String) = safeApiCall {
        val response = httpClient.get("/api/backtest/$strategyId")
        response.body<BacktestResultDto>().toDomain()
    }
}
```

## Troubleshooting

### Common Issues

#### Build fails with "Cannot find symbol: HttpClient"

**Solution**: Make sure you have the correct Ktor engine for your platform.

```kotlin
// In shared/build.gradle.kts
commonMain.dependencies {
    implementation(libs.ktor.client.core)
}

androidMain.dependencies {
    implementation(libs.ktor.client.okhttp)
}
```

#### Koin `NoBeanDefFoundException`

**Solution**: Make sure you called `initKoin()` before accessing dependencies.

#### API calls timeout

**Solution**: Check your base URL and network connectivity. Increase timeout if needed:

```kotlin
HttpClientFactory.create(
    baseUrl = "http://localhost:8000",
    timeoutMillis = 60_000  // 60 seconds
)
```
